# Handoff Order

## Planner

Own:

- problem framing
- repo research
- clarifying questions
- initial design or architecture direction

Output:

- plan document
- open questions
- cited sources

## Product Designer

Own:

- current-screen analysis
- UX/UI recommendations
- component mapping
- responsive and accessibility guidance

Output:

- design spec document

## Tasker

Own:

- plan audit
- missing-step detection
- dependency ordering
- execution task breakdown

Output:

- task document

## Implementers

Own:

- code changes
- tests
- implementation notes

Output:

- code, tests, summary
