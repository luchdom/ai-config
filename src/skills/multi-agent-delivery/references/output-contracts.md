# Output Contracts

## Before Tasker

Require:

- a concrete plan
- resolved or explicitly tracked open questions
- a design spec for non-trivial UI work

## Before Implementers

Require:

- a plan or task list with concrete scope
- likely files or modules
- acceptance criteria
- test expectations

## Before Final Review

Require:

- summary of what changed
- files touched
- known limitations or follow-ups
